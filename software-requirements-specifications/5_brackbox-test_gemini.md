# ブラックボックステスト仕様書 (Black Box Test Specification)
## 物理演算ベース・なぞり消しパズルゲームシステム 【確定版】

本ドキュメントは、要求仕様書（`srs_v2_final.md`）に定義された挙動・数値・ロジックに基づき、システムの内部構造（ソースコード）を参照せずに外部からの入力と出力のみで検証を行うためのブラックボックステスト仕様書である。

---

## 1. テスト設計の基本方針と採用技法

1. **同値分割法・境界値分析 (Equivalence Partitioning / Boundary Value Analysis)**
   * スコア計算ロジック（チェーン数 $N$）や、ユーザー名の文字数制限（1〜15文字）など、システムの挙動が切り替わる境界線を突くテストケースを設計する。
2. **状態遷移テスト (State Transition Testing)**
   * 各画面間の遷移、プレイ中から一時停止（ポーズ）へのステート切り替え、およびなぞり操作中のロールバックなど、状態管理（ステートマシン）の正しさを検証する。
3. **エラー推測法 (Error Guessing)**
   * 物理演算エンジン特有の「詰まり（スタック）」現象や、ネットワーク通信の遅延・タイムアウト、不正入力の試行など、バグや例外が発生しやすいエッジケースを狙い撃つテストケースを設計する。

### 2.1 スコア加算およびチェーン（なぞり消し）判定の検証
* **対象要件:** 4.2節（操作および消去ロジック）、4.2.3節（スコア加算ロジック）
* **数式基準:** $Score = \lfloor N \times 100 \times (1 + (N - 3) \times 0.1) \rfloor$

| テストケースID | テスト対象機能 | 入力・操作手順 | 期待される結果（判定基準） | テスト技法 |
| :--- | :--- | :--- | :--- | :--- |
| **TC-SC-01** | 最小チェーン未満 | 同一種類のオブジェクトを**2個**なぞって画面から指（ポインタ）を離す。 | ・オブジェクトは消去されないこと。<br>・選択状態が解除され通常状態に戻ること。<br>・スコアは `0` 点のままであること。 | 境界値分析 |
| **TC-SC-02** | 最小チェーン（境界値） | 同一種類のオブジェクトを**3個**なぞって画面から指を離す。 | ・オブジェクト3個が消滅し、補充が発生すること。<br>・スコアに **`300` 点** が即座に加算されること。 | 境界値分析 |
| **TC-SC-03** | 規定のチェーン | 同一種類のオブジェクトを**4個**なぞって画面から指を離す。 | ・オブジェクト4個が消滅し、補充が発生すること。<br>・スコアに **`440` 点** が即座に加算されること。 | 同値分割法 |
| **TC-SC-04** | ロングチェーン | 同一種類のオブジェクトを**10個**なぞって画面から指を離す。 | ・オブジェクト10個が消滅し、補充が発生すること。<br>・スコアに **`1,700` 点** が即座に加算されること。 | 同値分割法 |
| **TC-SC-05** | 操作のロールバック | オブジェクトを4個なぞった（ホールドした）状態のまま、なぞり経路を逆方向に辿り、**2番目になぞったオブジェクトの位置までポインタを戻して**離す。 | ・最後（4番目、3番目）になぞったオブジェクトが順次選択解除されること。<br>・離した瞬間のリスト数が2個以下のため消去が不成立（リセット）となり、スコアは加算されないこと。 | 状態遷移 |
| **TC-SC-06** | 非隣接オブジェクト | 選択中の末尾オブジェクトからの中心間距離が **120px（直径の1.2倍）を超える** 位置にある同種オブジェクト上へポインタを移動する。 | ・オブジェクトが選択リストに追加されない（反応しない）こと。<br>・ガイドラインも描画されないこと。 | エラー推測 |

### 2.3 物理ワールド配置および補充・詰まり防止の検証
* **対象要件:** 4.1.2節（物理パラメータ）、4.1.3節（初期配置）、4.1.4節（アンチ・スタックシステム）

| テストケースID | テスト対象機能 | 入力・操作手順 | 期待される結果（判定基準） | テスト技法 |
| :--- | :--- | :--- | :--- | :--- |
| **TC-PH-01** | 初期配置自動消滅防止 | ゲームを開始し、ステージ内に50個のオブジェクトが自由落下によって初期配置される状態を観察する。 | ・初期配置において同種が3つ以上隣接して噛み合っても、**勝手に消滅してスコアが入る現象が起きない**こと。<br>・プレイヤーが最初のタップ（`pointerdown`）を行うまで初期配置が維持されること。 | エラー推測 |
| **TC-PH-02** | オブジェクトの一括補充 | オブジェクトを10個なぞって消去し、消去エフェクト（アニメーション）が実行されている間のステージ内を観察する。 | ・消去中やなぞり操作の最中に、オブジェクトがパラパラとリアルタイム補充されないこと。<br>・消去エフェクトが**完全に終了した瞬間**に、不足分の10個が上部から一斉に生成されて落下してくること。 | 仕様準拠 |
| **TC-PH-03** | アンチ・スタック（詰まり防止） | 物理挙動によりオブジェクトが空中で噛み合い、下部に隙間があるにもかかわらず静止する状態を発生させ、3秒以上経過させる。※ | ・静止（速度0.1未満）から3秒経過した時点で、個体に微小なランダムフォース（揺れ）が加わること。<br>・噛み合わせが外れ、底面または他のオブジェクトの山まで自然に落下していくこと。 | エラー推測 |
| **TC-PH-04** | シャッフル機能（詰まり解消） | プレイ中に画面右上の「SHUFFLE（シャッフル）」ボタンをタップする。 | ・ステージ内のすべてのオブジェクトに対して、上向きの強い初期速度（`vy < 0`）とランダムな回転（角速度）が与えられること。<br>・オブジェクトが空中へ打ち上げられ、噛み合わせや配置が完全に再シャッフルされること。<br>・タイマーおよびスコアには影響を与えないこと。 | 仕様準拠 |

※ *注釈: テストの効率化のため、デバッグ環境等で一時的に摩擦係数（`friction`）を `1.0` などに高めてスタックを起こしやすくして確認することを推奨。*

## 3. テスト環境および実施上の推奨事項

* **ネットワーク例外のシミュレーション:**
  TC-API-08（タイムアウト）の検証は、Google Chrome等のデベロッパーツール（F12）の「Network」タブ内にある「Throttling」設定を `Offline` に切り替える、またはモックサーバーを用いてレスポンスを5.5秒遅延させることで、コードを変更することなくブラックボックス環境で容易に再現可能である。
* **物理テストの効率化:**
  TC-PH-03（アンチ・スタック）の動作検証は、自然発生を待つとテスト時間が肥大化するため、ステージの形状を一時的に狭める、あるいはオブジェクトの初期生成位置を1箇所に集中させるなどの「テスト用配置（テストハーネス）」を用意して実施することが望ましい。
